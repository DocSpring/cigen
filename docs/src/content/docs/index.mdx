Cigen generates CircleCI configuration from a concise, provider-agnostic format, with strong validation and sensible conventions. Other providers are planned but not yet supported.

## Why Cigen?

<CardGrid>
  <Card title="Validated Config" icon="puzzle">
    Write your pipeline in a small, reusable format. Cigen validates it and emits native CircleCI YAML.
  </Card>
  <Card title="Multi-Architecture Support" icon="laptop">
    Automatically generate jobs for multiple architectures (amd64, arm64) from a
    single job definition.
  </Card>
  <Card title="Cache Helpers" icon="rocket">
    Injects restore/save steps when caches are declared on a job. Definitions are extensible.
  </Card>
  <Card title="Advanced Checkout" icon="git-branch">
    Shallow git checkout by default with configurable clone/fetch options, host key scanning, and path.
  </Card>
  <Card title="Pragmatic Roadmap" icon="star">
    Advanced features like OR-deps and approval status fixes are planned.
  </Card>
</CardGrid>

## Core Philosophy

**If we need a feature the platform lacks, we'll add safe workarounds—with clear scopes and validation.**

## Quick Example

Define your pipeline structure:

```
.cigen/
├── cigen.yml                    # Provider and global config
├── config/
│   └── source_file_groups.yml  # Reusable file patterns
├── workflows/
│   └── main/
│       └── jobs/
│           ├── test.yml         # Test job definition
│           └── build.yml        # Build job definition
```

```yaml title=".cigen/config.yml"
provider: circleci
```

```yaml title=".cigen/config/source_file_groups.yml"
source_file_groups:
  nodejs:
    - 'src/**/*.js'
    - 'package.json'
    - 'package-lock.json'
```

```yaml title=".cigen/workflows/main/jobs/test.yml"
image: cimg/node:18.0
source_files: '@nodejs'
packages: node
steps:
  - run: npm test
```

```yaml title=".cigen/workflows/main/jobs/build.yml"
image: cimg/node:18.0
architectures: [amd64, arm64]
requires: [test]
source_files: '@nodejs'
packages: node
steps:
  - run: npm run build
```

Generate CircleCI configuration:

```bash
cigen generate
```

Results in a `.circleci/config.yml` with:

- Git checkout
- Optional package installation steps (npm/yarn/pnpm/bun, bundler, etc.)
- Separate jobs for each architecture (`build_amd64`, `build_arm64`)
